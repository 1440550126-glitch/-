# AI句灵 · 让每一句话活过来

面向年轻人的 **AI 文案社交圈**：发一句话 → AI 生成专属预览卡 → 长按让文字**活过来**（非线性线条动画 + 实时合成音效），配上 AI 暖场官、轻量桌游大厅（谁是卧底 · AI 主持）、皮肤商城与 9.9 元会员的完整商业闭环。

> 产品定位、功能取舍、风险判断见 [docs/PRODUCT.md](docs/PRODUCT.md)（建议先读）。

## 一键运行（零依赖）

只需要 **Node.js ≥ 22.5**（内置 SQLite / fetch），不需要 `npm install`，不需要数据库服务：

```bash
npm start
# 📱 用户端    http://localhost:3000
# 🛠 管理后台  http://localhost:3000/admin   （默认 admin / jvling-admin-2026）
```

默认**不配置任何大模型 Key 也能完整体验**：所有 AI 能力（预览卡 / 动画导演 / 暖场 / 话题 / 主持）自动落到本地规则引擎，成本为 0。接入大模型只需复制 `.env.example` 为 `.env` 填入任意 OpenAI 兼容接口（火山方舟 / 阿里百炼 / 腾讯混元 / DeepSeek）。

```bash
npm run smoke            # 126 项全链路冒烟测试（社交+审核+对局 + 灵阵 AI 团队全链路）
node scripts/llm-test.mjs  # 大模型连通性一键自检（配好 .env 后运行）
npm run dev     # 开发模式（文件变更自动重启）
```

## 已实现功能

**文案社交圈**：发布（敏感词三级审核）、推荐/最新/关注/热门四种信息流、点赞、评论与楼中楼回复、收藏、分享、关注、个人主页、今日话题（AI 生成 · 带标识）。

**文字变动画（核心卖点）**：服务端大模型当"导演"产出 Animation Manifest（情绪/场景/角色/粒子/时间轴/声音），客户端 Canvas 引擎做"播放器"——文字粒子化重组、火柴人行为系统（等待/奔跑/拥抱/仰望…）、风雨雪樱星萤等粒子、情绪参数调制速度与微动、Web Audio 实时合成音效（风声/雨声/心跳/碎裂，零音频资源）。免费 3 次/天体验，会员解锁樱粉/霓夜/星海风格，高级风格走星尘额度。

**AI 暖场系统**：3 个人设账号（小句灵/灵感官/主持官），自动发帖（每日 10-30 条可配）、延迟 30s-10min 评论用户新帖（每帖 ≤2 条）、每日话题、大厅组局提醒（仅大厅无房时）、夜间自动降频、每日成本预算封顶、用户可关闭、后台一键全关。**所有 AI 内容带标识，AI 点赞单独计数不进热度（不伪造热度）**。

**桌游大厅**：游戏引擎框架（房间/座位/AI 主持/AI 陪练/计时器为共享底座）+ 两款完整玩法——**谁是卧底**（发词/轮流描述/投票/复盘）与**狼人杀**（狼人/预言家/女巫/平民，天黑行动→白天发言投票完整循环，夜间行动走私有通道、狼队私聊、查验记录、女巫药剂管理），SSE 实时同步，AI 陪练补位带明确标识，一个人也能开局。剧本杀/密室逃脱排期见 ROADMAP。

**🛰 灵阵 · AI 团队（多智能体协作平台 · 对标并超越扣子）**：把任务交给一支会分工的 AI 团队——**编排官**拆解任务、按专长把活派给最合适的成员，**各专家成员**带工具（计算器 / 知识库检索 / 网页抓取 / 文本统计 / 日期 / 随机）跑 ReAct 循环、共享"黑板"记忆产出，**总编**把各方产出整合成可直接交付的成果；全程 SSE「作战室」实时直播每一步在干什么。四种协作策略（**编排协作 / 流水线 / 智能路由 / 圆桌论战**，论战为真·多轮互评）、可视化团队工作台、7 个内置智能体 + 4 个团队模板开箱即用、知识库 **RAG**（零依赖关键词检索，中文 bigram 倒排打分，可平滑升级向量）。团队可**发布到「团队广场」**供他人一键调用；**后台**提供运行监控、强制停止与每日预算管控。**沿用全局铁律：无大模型 Key 时全程走本地引擎，真实调用工具、产出结构化成果，零成本可完整体验**；运行成本计入 AI 日报、受每日预算封顶与每日运行配额约束（免费 8 次 / 会员 80 次）。设计细节见 [docs/AGENTS.md](docs/AGENTS.md)。

## 技术架构

```
┌── web/      移动端 H5 App（原生 ES Modules，零构建）
│     ├ js/anim/   动画引擎 + Web Audio 合成 + 播放器
│     ├ js/fx/     WebGL 流体背景（fbm 噪声 shader）
│     └ js/pages/  12 个页面
├── admin/    管理后台 SPA
├── server/   Node 22 零依赖服务端
│     ├ lib/       db(node:sqlite) / 鉴权 / 审核 / LLM 客户端 / Manifest 导演引擎
│     ├ routes/    70+ REST 接口 + SSE
│     ├ game/      游戏框架 core + 卧底/狼人杀双引擎
│     ├ agents/    灵阵：多智能体编排引擎 + 工具/插件 + 知识库(RAG) + 模板
│     └ warmup/    AI 暖场调度器
├── miniprogram/  微信小程序版（历史版本，可作为渠道端）
└── docs/     产品定位 / API / 数据库 / 合规
```

技术决策的理由（为什么不是 Flutter+NestJS+PG 起步）见 [docs/PRODUCT.md](docs/PRODUCT.md) 第三节；Animation Manifest 协议与端无关，未来 Flutter 客户端按协议实现渲染器即可复用全部服务端。
