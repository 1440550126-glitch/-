# 代码审阅

1. **读取**：用 `read_file` 看目标文件；必要时 `list_dir` 了解结构、`run_shell` 跑测试/复现。
2. **分层审阅**，按严重度从高到低：
   - 正确性 / 边界：空值、越界、并发、资源未释放、异常吞掉
   - 安全：注入、路径穿越、密钥硬编码、反序列化、SSRF
   - 可维护性：命名、重复、过长函数、缺测试
   - 性能：明显的 O(n²)、重复 IO、N+1
3. **给可执行修复**：对关键问题给出最小补丁（before/after 片段），解释为什么。
4. **验证**：能跑就用 `run_shell` 跑测试/静态检查确认修复，不臆断"应该没问题"。
5. **输出**：先列 Top 问题清单（按严重度+置信度），再给补丁与验证结果。

匹配周围代码风格，不顺手改无关内容；不确定处明确标注并给出排查建议。
