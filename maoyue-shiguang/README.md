# 猫约时光 · 情侣陪伴版（v3）

一个「情侣陪伴 + 共同小金库」的微信小程序：和 TA 一起记录在一起的日子、养猫、攒钱、收藏回忆。**纯情侣使用，无陌生人社交。**

## 功能总览

**底部 5 个 Tab：家 / 陪伴 / 时光 / 金库 / 我的**

| 模块 | 路径 | 说明 |
|------|------|------|
| 家 | `pages/home` | 在一起天数、一键传情、今日心情、今日一问、入口（猫窝/时光/点餐/会员）、未绑定时引导邀请 TA |
| 陪伴 | `pages/interact` | 戳一戳、心情同步、每日一问、爱的小纸条 |
| 时光 | `pages/timeline` | 纪念日倒数、共同愿望、时光轴回忆 |
| 金库 | `pages/vault` | **共同小金库**：存钱/记账、投币动画+音效、刷卡动画+音效、开库金光第一人称动画+音效 |
| 我的 | `pages/profile` | 关系绑定、**主题切换**、会员中心、实名认证、**法律条款**、设置 |
| 猫窝 | `pages/cat` | 一起养的猫（免费、靠互动喂养） |
| 点餐 | `pages/dining` | "今天吃什么"随机转 + 自定义菜单 |
| 状态 | `pages/status` | TA 的电量/在线/位置概览（地图距离，原型为模拟） |
| 相册 | `pages/album` | 回忆相册 |
| 会员 | `pages/membership` | 会员套餐与权益（原型：模拟开通） |
| 实名 | `pages/verify` | 实名认证（原型：模拟核验；纯情侣 app 非必需，可移除） |
| 法律 | `pages/legal` | 隐私政策 / 用户协议 / 社区规范 / 未成年人保护（顶部可切换） |

## 主要功能

- **🎨 主题切换**：珊瑚粉 / 奶咖 / 莫兰迪 / 马卡龙 / 暗夜，全局换肤（「我的 → 主题皮肤」）。
- **🍜 情侣点餐**：随机决定"今天吃什么"，菜单可自定义增删。
- **💰 共同小金库**：两人共同的存钱/记账本，绑定后互通（存钱掉金币 / 记花销刷卡 / 开库金光，均带音效）。
- **🐱 一起养猫 + 🕰 时光轴 + 🎂 纪念日 + 📷 回忆相册**。
- **🔗 情侣双向同步**：邀请码绑定后，愿望/纪念日/回忆/猫/小金库/菜单 + 心情/纸条/每日一问在两台手机间自动同步（离线优先，云开发；按作者区分，self/partner 不对调）。
- **📡 准实时 + 在场状态**：对方改动约 10s 内可见（watch 优先、轮询兜底）；TA 的电量/在线/位置真实上报（位置默认不共享），**戳一戳**实时送达。
- **👑 会员体系 + 💳 微信支付**：云函数 `pay` 下单付款（未配置商户时自动模拟开通）。
- **🔒 内容安全**：纸条 / 愿望 / 昵称等输入经云函数 `secCheck`（`msgSecCheck`）校验。
- **📜 法律条款**：隐私政策 / 用户协议 / 社区规范 / 未成年人保护，「我的 → 设置」可查看。

## 云开发 / 后端

- `app.js` 已加云开发初始化；在 `app.js` 顶部填 `CLOUD_ENV`（云开发环境 ID，留空用默认环境）。
- `cloudfunctions/coupleData/`：情侣绑定 + 共享数据双向同步 + 准实时 `ping`（云端权威合并）。客户端在 `utils/cloud.js` + `utils/store.js` 同步层。**详见 `docs/CLOUD-SYNC.md`**。
- `cloudfunctions/secCheck/`：内容安全（`msgSecCheck`/`imgSecCheck`），封装 `utils/seccheck.js`。
- `cloudfunctions/pay/`：微信支付下单 + 回调发放会员，封装 `utils/pay.js`。**详见 `docs/PAYMENT.md`**。
- 三个云函数右键「上传并部署：云端安装依赖」；需建集合 `couples`、`users`。**未开通云开发时全部自动降级**（同步跳过、内容安全放行、支付走模拟），本地照常体验。

## 音效

`assets/audio/` 下的 `coin.wav` / `swipe.wav` / `vault.wav` / `cine.wav` 为**代码合成**的占位音效，可替换成成品（保持文件名即可）。

## 如何运行

1. 微信开发者工具 →「导入项目」→ 选本目录。
2. 编译即可；「预览」生成二维码真机体验。
3. 如需内容安全/云端：先开通云开发并部署 `secCheck`（见 `docs/UPLOAD-GUIDE.md`）。

## 原型说明与合规提示（重要）

**未开通云开发时**数据存本地缓存（`utils/store.js`），无需后端即可体验；TA 的状态/心情/答案为温柔模拟。**绑定 + 开通云开发后**：双向同步、准实时、在场状态/戳一戳、心情/纸条/每日一问按作者同步全部生效（详见 `docs/CLOUD-SYNC.md`）。
- **小金库**是"记账本"，金额由用户手动记录；微信支付仅用于会员，未配置商户时为模拟开通。
- 上线部署：开通云开发 → 建集合 `couples`/`users` → 部署 `coupleData`/`secCheck`/`pay` 三个云函数；微信支付另需商户号配置（见 `docs/PAYMENT.md`）。
- 实名认证仍为原型（纯情侣 app 非必需，可移除）。
- 已移除全部陌生人社交，**不再需要「婚恋交友」类目资质**。
- 上架步骤见 `docs/UPLOAD-GUIDE.md`，隐私后台填写见 `docs/PRIVACY-GUIDE.md`，法律文本见 `docs/legal/`。
