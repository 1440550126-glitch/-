# 小程序后台「用户隐私保护指引」填写对照表

> 微信要求：凡收集用户信息的小程序，必须在 **mp.weixin.qq.com → 设置 → 服务内容声明 → 用户隐私保护指引** 中逐项声明"收集什么 + 用途"，并配置《隐私政策》链接。**未配置会被审核驳回，且调用 `getLocation`、`chooseMedia` 等接口会报错。**
> 下表是按"猫约时光"**当前实际行为**整理的勾选清单，照着填即可。

## 一、需要勾选声明的"收集信息"项

| 后台信息类型 | 是否收集 | 对应接口 | 用途说明（可直接填） |
|---|---|---|---|
| 位置信息 | ✅ 是 | `wx.getLocation` | 用于在地图上显示你与 TA 之间的距离 |
| 相册（拍摄/选择媒体） | ✅ 是 | `wx.chooseMedia` | 用于你添加"回忆"的照片或视频 |
| 设备信息 | ✅ 是 | 系统/日志 | 用于保障小程序正常运行、安全与问题排查（崩溃日志等） |
| 微信昵称、头像 | ⬜ 否 | 用户手动填写，不从微信读取 | （昵称由用户输入、头像为内置表情，**不获取**微信资料，可不勾） |
| 微信账号（OpenID） | ⚠️ 启用云开发后勾 | 云开发 / `wx.login` | 用于识别账号、云端同步，以及内容安全 `secCheck` 校验（不获取微信密码） |
| 手机号 | ⬜ 否 | 未使用 | — |
| 身份证/实名信息 | ⬜ 否 | 实名为原型，**未真实收集** | 纯情侣 app 非必需；如保留并接真实核验，应交由持牌渠道处理 |
| 通讯录/麦克风/剪贴板 | ⬜ 否 | 未使用 | — |

> 说明：开通**云开发**并部署 `secCheck` 后，云函数会用到你的 OpenID（用于内容安全校验与云端同步），届时请勾选并声明"微信账号(OpenID)"。未开通云开发的纯本地阶段保持未勾选，以与实际行为一致（审核很看重"声明与实际一致"）。

## 二、配套要做的两件事

1. **隐私政策链接**：后台"用户隐私保护指引"里需要填写《隐私政策》。
   - 小程序内已内置：**我的 → 设置 → 隐私政策 / 用户协议**（页面 `pages/legal/legal`）。
   - 如后台要求一个 **URL**，把 `docs/legal/隐私政策.md` 的内容放到你的官网/备案域名页面，填该网址。
2. **地理位置接口申请**：`app.json` 已声明 `requiredPrivateInfos: ["getLocation"]`。
   还需在 **开发管理 → 接口设置** 申请"地理位置"接口权限（填用途：显示与 TA 的距离）。企业主体可申请。
