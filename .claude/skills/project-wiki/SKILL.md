# Project Wiki Skill — 自维护知识库

知识库位于仓库根的 `wiki/` 目录，`wiki/index.md` 是总索引。生命周期三步：

## 1. 查询 Query（每次开工先做）
- 动手前先读 `wiki/index.md`，再读相关页（如改供应商先读 `wiki/providers.md`）。
- 回答/改动要**标注来源页**；资料不足就先去看真实代码，并记下"该补哪一页"。
- 运行时也有同一套知识：App 内置 `wiki_query`（MCP/Agent 工具）查 `wiki_pages` 表。

## 2. 摄入 Ingest（收尾时做）
- 新增/改了重要能力（供应商、一致性手法、模型、部署、接口）后，**更新对应 wiki 页**：
  - 提炼要点、写清"是什么/在哪/怎么用"，不要贴大段代码；
  - 更新 `wiki/index.md` 索引与交叉引用（`[[页名]]` 风格）；
  - 一个主题一页，避免重复页。

## 3. 审计 Audit（定期/大改后做）
- 通读 index 与各页：找**矛盾声明、孤立页（没人引用）、空缺（该有却没有的页）、过期内容**。
- 发现矛盾以**真实代码为准**修正；孤立页并入相关页或补引用。

## 与运行时知识库的关系
- `wiki/`（本目录）= 给"写代码的 Agent"看的工程知识；
- App 的 `wiki_pages` 表 = 给"创作工作流/产品 Agent/外部音乐站"用的创作知识（domain 分区）。
- 两者范式相同、各司其职；改了 App 的 wiki 能力，记得同步本目录 `wiki/llm-wiki.md`。
