# 句灵 · 产品定位决策书

（对应需求文档第二十二节"请你最终输出什么"的 1-7 项）

## 一、产品定位：选方向 C，但重心明确偏向文案社交

**定位一句话：年轻人的"文字活过来"社交圈 —— AI 文案社交为主体，轻量桌游为留存抓手，会员+皮肤+高级额度商业化。**

为什么不是方向 A（纯文案社交）：缺少"用户停留场所"，发完即走，冷启动留存差，AI 暖场官没有舞台。

为什么不是方向 B（桌游为主）：实时游戏的开发/运维/审核成本是文案社交的数倍；重游戏化形态有版号认定风险；桌游本身无差异化（市面已有头部产品），拼不过。

方向 C 的取舍：文案社交 + 文字变动画承担**差异化与传播**（人无我有），桌游大厅承担**关系链与时长**（让认识的人有地方一起玩），AI 暖场承担**冷启动氛围**。三者共享同一套账号/审核/商业化底座，边际成本低。

## 二、第一版保留什么、砍掉什么

### 保留（已全部实现）
1. 文案社交全链路（发布/四种流/点赞/评论/回复/收藏/分享/关注/主页/话题）
2. AI 预览卡（零成本规则引擎，人人免费）
3. 文字变动画（核心卖点，做深：状态机/行为/情绪参数/粒子/合成音效）
4. AI 暖场官（频控/夜降/预算/可关/全标识）
5. 桌游大厅 + **谁是卧底一个游戏做完整**（含 AI 主持与 AI 陪练）
6. 9.9 会员 + 皮肤商城 + 星尘额度（沙盒支付完整闭环）
7. 审核/举报/拉黑/封禁/注销 + 管理后台 + AI 成本统计

### 砍掉（及理由）
- **狼人杀、剧本杀、密室逃脱**：砍到 ROADMAP。三个半成品不如一个完整品；谁是卧底已验证了房间/实时/AI 主持的全部基础设施，后续游戏是在同一状态机框架上加规则，而不是推倒重来。剧本杀/密室还强依赖内容生产管线（剧本库），第一版做必然空壳。
- **语音房**：实时音频需要 RTC 供应商接入与内容安全审计（语音审核成本极高），且不影响核心验证。座位/房间结构已为其预留。
- **真视频生成**：成本不可控、延迟差、审核风险高。Manifest 方案在体验上反而更"活"且可循环播放。
- **联名皮肤、广告**：商业化第四、五优先级，等有量再谈。
- **手机号/微信登录**：需要短信与开放平台资质，先游客+账号密码，接口已预留升级位。
- **Unity 3D 房间**：第一版用 WebGL 流体 + Canvas 粒子达成"伪 3D 流体"视觉，成本 1/100。

## 三、技术选型（与建议方案的差异及理由）

| 建议 | 实际采用 | 理由 |
|---|---|---|
| Flutter App | **移动端 H5（零构建原生 JS）** | 文字变动画的灵魂是 Canvas/WebGL/WebAudio，H5 全部原生支持且即点即用，便于快速分发验证（可包 WebView 上架）。Animation Manifest 协议与端无关，验证成功后 Flutter 按协议实现渲染器即可，服务端零改动。仓库内另有微信小程序版可作渠道端。 |
| NestJS | **Node 22 零依赖 http** | MVP 阶段 55 个接口用微框架 600 行搞定，无依赖供应链风险，`npm start` 即跑。业务长到需要团队协作时再迁 NestJS，路由层已按模块切分，迁移是搬运而非重写。 |
| PostgreSQL | **node:sqlite（内置）** | 零安装跑通是第一优先级。表结构按 PG 习惯设计（见 DATABASE.md），日活上万再迁。 |
| Redis | 进程内存（限流/房间态） | 单实例阶段 Redis 是负优化；Hub/限流已抽象成独立模块，集群化时替换实现即可。 |
| WebSocket | **SSE + POST** | 回合制桌游与聊天的下行实时性 SSE 完全够用，自动重连、过代理友好、零依赖；上行走 POST 顺带复用全部审核/限流管线。 |
| LLM 多供应商 | 统一 OpenAI 兼容客户端 | 火山/百炼/混元/DeepSeek 全部兼容此协议，换供应商改两行环境变量。**Key 只在服务端**，所有调用记 token 与成本（微元精度），失败自动落本地规则引擎。 |

## 四、主要风险与应对

1. **内容安全（最大风险）**：UGC + AI 生成双重审核义务。已做三级敏感词（可热更）+ 人工审核队列 + 全量 AI 标识 + 自伤关怀流程；上线前需接入云厂商内容安全 API（文本反垃圾）做机审第一道。
2. **AI 成本失控**：已做每日配额、按用户限流、暖场每日预算封顶（超限自动降级本地模板）、成本逐条记账后台可视。默认模型用 flash 级低价模型。
3. **版号认定**：谁是卧底以"社交房间内的互动玩法"形态存在，无积分排位、无虚拟货币消耗、无数值成长，皮肤不进游戏逻辑。若监管口径收紧，可一键下线桌游模块（独立路由）而不伤社交主体。
4. **支付合规**：现为沙盒支付（订单/履约/记账真实）。iOS 虚拟商品必须走 IAP；安卓走微信/支付宝小微或企业资质。未成年人：青少年模式服务端硬拦截消费。
5. **冷启动**：AI 暖场官 + 种子内容已解决"第一屏空荡"；但 AI 不能替代真实关系，运营侧仍需校园/兴趣社群冷启动策略。
6. **抄袭与同质化**：文字变动画的壁垒在"导演引擎 + 端侧渲染协议"的持续迭代速度，需保持每月上新动画风格/元素的节奏（皮肤系统天然承接）。

## 五、商业模式（优先级与红线）

收入结构：① 9.9 元/月会员（主）→ ② 皮肤（1.9-98 元五档，情绪价值定价）→ ③ 星尘额度（高级 AI 按次，会员 8 折，生成前明示消耗）→ ④ 联名/广告（后期）。

红线（已在代码层面强制）：不卖胜负/身份/投票权（游戏逻辑读不到任何付费状态）、无抽奖无赌博、AI 不诱导消费（人设 prompt 明令禁止）、青少年模式禁购、AI 点赞不计热度。

## 七、开发路线图

- **v0.1（本版）**：上文"保留"清单全部，73 项冒烟 + 浏览器 E2E 通过。
- **v0.2**：云厂商内容安全机审接入、微信/支付宝支付、手机号登录、消息通知中心、文字变动画分享为短视频（端侧录制 Canvas）。
- **v0.3**：狼人杀（复用房间状态机，新增夜晚阶段与角色技能）、海龟汤（AI 主持天然适配）、房间语音（RTC）。
- **v0.4**：剧本杀（先上 AI 即兴短剧本模式降低内容成本）、Flutter 客户端、PostgreSQL + Redis 迁移、推荐算法升级。
