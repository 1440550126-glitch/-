# 广告变现配置

本项目收入主要来自微信「流量主」广告。本文讲怎么开通、配广告位、以及收益预期与调优。

## 一、开通「流量主」的门槛

- 小程序**累计独立访客（UV）≥ 1000**。
- 类目符合流量主准入（工具类一般可以）。
- 无严重违规记录。

> 所以发布初期不会有广告收入，先靠真实用户把 UV 做到 1000。可通过分享、社群、短视频引流冷启动。

## 二、创建广告位

[mp.weixin.qq.com](https://mp.weixin.qq.com) → **流量主** → 广告管理 → 新建广告位：

| 广告位类型 | 用在哪 | 对应 config 字段 |
| --- | --- | --- |
| 激励视频 | 结果页「保存」前播放（核心收入） | `rewardedAdUnitId` |
| Banner | 首页 / 结果页底部 | `bannerAdUnitId` |
| 插屏 | 保存成功后偶发弹出 | `interstitialAdUnitId` |

把生成的 `adunit-xxxx` 填进 `miniprogram/config.js`。

## 三、变现策略（在 config.js 调）

```js
requireAdToDownload: true,   // 必须看完激励视频才能下载（收入主来源，建议 true）
freeDownloadsPerDay: 0,      // 每天免费下载次数；0=每次都看广告；设 1~2 可提升留存
interstitialAdUnitId: '',    // 插屏；体验敏感，建议低频或留空
```

权衡：

- `requireAdToDownload=true` + `freeDownloadsPerDay=0`：变现最强，但新用户体验偏硬，可能影响留存与口碑。
- 给 1~2 次免费额度：第 1 次不看广告直接存，培养习惯，之后再用广告墙——通常 **LTV 更优**。
- 插屏别滥用，密集插屏会拉低评分。

## 增长玩法：新人免广告 + 分享得次数

由 `quota` 云函数实现，按 openid 服务端记额度（防本地刷量），客户端不可用时自动降级。

- **新人前 N 次免广告**（`NEWBIE_FREE`，默认 3）：新用户进来就能直接保存几次、不被广告劝退，先体验到价值，留存更好。
- **分享得次数**（`SHARE_REWARD`/次，默认 2；每日上限 `SHARE_DAILY_CAP`，默认 3）：用户分享小程序即 +免广告次数，形成「分享→拉新→更多用户」的增长循环。
- **每日免费**（`DAILY_FREE`，默认 0）：可给每天 1~2 次保底免费，提升日活粘性。

下载时的扣减优先级：**每日免费 → 免广告额度 → 看广告**。免广告路径保存失败会自动退还额度。

**取舍**：免广告次数会**短期减少广告曝光**，但通常换来更高留存与分享拉新，把 DAU 做大，长期总收入更高。建议：

- 冷启动期把 `NEWBIE_FREE` 调大（如 5）、`SHARE_REWARD` 给足，优先做规模。
- 上量后再收紧（如 `NEWBIE_FREE=2`），平衡 eCPM 收入与体验。
- `SHARE_DAILY_CAP` 别给太高，防止刷分享白嫖。

> 「分享得次数」在分享动作触发时发放（每日封顶）。微信无法回调"好友是否真的打开"，故以"触发分享 + 每日上限"控制成本。

### 邀请裂变（真·拉新归因）

在「分享得次数」之上，再做了**基于邀请进入的归因**，奖励与"真的拉来新用户"挂钩：

- 分享链接自动带上分享者 openid（`/pages/index/index?ref=openid`）。
- 新用户从链接进入、完成登录后，客户端调用 `quota` 的 `bind`：**邀请人 +`REF_REWARD`（默认 5）次，被邀请新人额外 +`REF_NEWBIE_BONUS`（默认 2）次**。
- **防刷四重**：① 一次性绑定（每个账号只能被归因一次）；② 不能自己邀请自己；③ 仅 `REF_BIND_WINDOW_H` 小时内创建的新账号可被归因（老用户点链接无效）；④ 邀请人每日奖励数封顶 `REF_DAILY_CAP`。
- 「我的」页展示"已成功邀请 N 人"，给用户正反馈、促持续分享。

裂变是这类工具型小程序低成本起量的核心引擎：奖励直接换成"少看几次广告"，对用户有真实价值，边际成本约等于让出的广告曝光。建议冷启动期把 `REF_REWARD` 给足，跑通"邀请→注册→留存"的循环后再调优。

## 四、收益预期（粗略）

激励视频 eCPM 受类目、地区、季节影响大，**仅供量级参考，不构成承诺**：

- 激励视频 eCPM 常见区间约 **¥30~¥80 / 千次有效播放**（高度波动）。
- 估算：日活 1000 × 人均 1.5 次完整激励视频 = 1500 次/天 → 约 **¥45~¥120 / 天**。
- 实际取决于完播率、填充率、用户地域与活跃度。

提升收入的杠杆：**做大 DAU**（引流）＞ 提高人均解析次数（产品粘性）＞ 调广告策略。

## 五、对账与数据

- 流量主后台看广告收入、eCPM、填充率。
- 本项目 `parse_logs` 集合记录每次解析（平台/来源/类型），可在云开发控制台或自建后台统计解析量、留存。
- 关注**完播率**：激励视频中途退出不计收入，`utils/ad.js` 已对"未看完"不予解锁。

## 六、合规提醒

- 只能用官方流量主广告位，禁止接第三方广告 SDK。
- 不得诱导点击广告、不得用广告遮挡内容误导点击。
- 广告频次与体验会进入小程序质量评估，别为短期收入牺牲体验。
- 详见《微信小程序流量主协议》与运营规范。
